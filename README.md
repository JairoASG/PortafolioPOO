# Descripción General

Este repositorio reúne las evidencias y proyectos desarrollados durante la asignatura de Programación Orientada a Objetos (POO) de la carrera de Ingeniería Informática en el Instituto Tecnológico Superior de Lerdo.

Los proyectos fueron diseñados para aplicar y fortalecer los principios fundamentales de la programación orientada a objetos, tales como la abstracción, encapsulamiento, herencia, polimorfismo, manejo de excepciones y persistencia de datos. Además, permiten desarrollar habilidades relacionadas con el diseño de software, la organización de código y la resolución de problemas mediante herramientas computacionales.

# Tecnologías Utilizadas
Lenguajes de programación: PHP y Python
Paradigma principal: Programación Orientada a Objetos (POO)
Desarrollo web: HTML5 y CSS3
Gestión de datos: Archivos de texto y sesiones de servidor
Herramientas complementarias: Sistema de archivos local y estructuras de datos dinámicas
Competencias Desarrolladas
Diseño y modelado de soluciones mediante clases y objetos.

Separación entre lógica de negocio e interfaz de usuario.
Validación y protección de datos ingresados por el usuario.
Manejo de archivos y almacenamiento de información persistente.

Desarrollo de aplicaciones estructuradas siguiendo buenas prácticas de programación.
Proyectos Desarrollados
# 1. Analizador de Consumo Energético en Servidores

Tecnologías: PHP, Namespaces, HTML y CSS.

# Descripción:
Aplicación web orientada al análisis del consumo energético en servidores. Permite calcular la energía consumida utilizando métodos numéricos de integración para estimar valores a partir de funciones de potencia variables.

# Conceptos Aplicados
Abstracción: El proceso matemático se encuentra encapsulado dentro de una clase especializada.
Encapsulamiento: Los parámetros de cálculo permanecen protegidos mediante atributos privados.
Organización del código: Uso de espacios de nombres para mejorar la estructura del proyecto.
Manejo de excepciones: Validación de datos y control de errores durante la ejecución.
# 2. Sistema Integral de Gestión de Usuarios y Permisos

Tecnologías: Python y Programación Orientada a Objetos.

# Descripción:
Sistema de consola diseñado para administrar distintos tipos de usuarios con niveles de acceso específicos, permitiendo una gestión organizada de permisos y privilegios.

# Conceptos Aplicados
Herencia: Implementación de clases derivadas a partir de una clase base.
Uso de constructores heredados: Aplicación de super() para reutilizar funcionalidades.
Polimorfismo: Personalización de comportamientos según el tipo de usuario.
Encapsulamiento: Validación y protección de información mediante propiedades controladas.
# 3. Sistema Web de Gestión de Bitácoras

Tecnologías: PHP y manejo de archivos de texto.

# Descripción:
Aplicación web destinada al almacenamiento y consulta de registros utilizando archivos de texto como medio de persistencia de información.

# Conceptos Aplicados
Persistencia de datos: Lectura y escritura de información en archivos.
Sanitización de entradas: Protección frente a contenido malicioso ingresado por el usuario.
Gestión de recursos: Apertura y cierre controlado de archivos para evitar errores de escritura.
# 4. Actividad Integradora: Control de Inventarios Dinámicos

Tecnologías: PHP, sesiones de servidor y arreglos.

# Descripción:
Sistema orientado a la administración de productos, permitiendo registrar artículos, calcular estadísticas y generar reportes básicos de inventario.

# Conceptos Aplicados
Estructuras de datos: Uso de arreglos para almacenar y relacionar información.
Persistencia mediante sesiones: Conservación de datos durante la navegación del usuario.
Funciones nativas de PHP: Empleo de herramientas integradas para realizar cálculos y análisis de información.
# 5. Examen Práctico: Modelado de Datos Estrictos

Tecnologías: PHP 8.

# Descripción:
Actividad realizada bajo restricciones de tiempo enfocada en la creación de clases correctamente estructuradas, aplicando principios de encapsulamiento y tipado de datos.

# Conceptos Aplicados
Tipado estricto: Declaración explícita de tipos de datos para propiedades y variables.
Control de acceso: Uso adecuado de modificadores de visibilidad para proteger la información interna de los objetos.
Modelado orientado a objetos: Representación de entidades mediante clases con atributos y comportamientos definidos.
