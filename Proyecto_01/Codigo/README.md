Nombre del proyecto: FastDelivery & Sensor System (Examen Práctico Corte 1)

Objetivo del proyecto:
Evaluar la precisión en la sintaxis de PHP y la capacidad para modelar datos mediante la creación de clases y propiedades con tipado estricto, sin implementar métodos o comportamientos dentro de las clases.

Problema que resuelve:
El proyecto permite organizar información relacionada con dos áreas operativas:

Logística: Se diseñó una estructura de datos para almacenar y dar seguimiento a la información de paquetes de envío.

Monitoreo: Se creó un modelo para representar sensores de plantas, registrando sus lecturas y los rangos técnicos de funcionamiento.

Tecnologías utilizadas:

Lenguaje de programación: PHP 8.x con tipado estricto.
Servidor local: XAMPP / Apache.
Editor de código: Visual Studio Code (VS Code).

Conceptos aplicados:
Basados en los temas de Programación Orientada a Objetos:

Creación y definición de clases.
Uso de modificadores de acceso (public y private).
Declaración de propiedades con tipado estricto.
Creación e instanciación de objetos.
Uso de clases predefinidas como DateTime.
Inclusión de archivos mediante require_once.

Reflexión personal

¿Qué aprendí?
Aprendí a utilizar el tipado estricto en PHP, lo que ayuda a evitar errores desde la definición de las clases y mejora la organización de los datos. Además, reforcé el uso de clases nativas como DateTime dentro de proyectos orientados a objetos.

¿Qué fue difícil?
Lo más desafiante fue cumplir con el tiempo limitado de la práctica mientras mantenía la estructura de carpetas solicitada y verificaba que la sintaxis de las propiedades tipadas fuera correcta.

¿Qué mejoraría?
Si el proyecto continuara desarrollándose, agregaría constructores para inicializar las propiedades de una forma más práctica y también implementaría métodos getters y setters para administrar correctamente la propiedad privada costoInterno.
